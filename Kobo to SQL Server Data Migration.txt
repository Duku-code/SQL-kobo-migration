# Kobo to SQL Server Data Migration Pipeline (Flexible Version)

## Purpose
This tool helps you interactively move KoboToolbox survey data into any SQL Server database (cloud or on-premise), letting you:
- Choose the target database on a server
- Specify your own table name
- Generate a CREATE TABLE script you can review, copy, and execute in SQL Server Management Studio (SSMS) or Azure Portal

---

## Prerequisites
- **ODBC Driver 17 for SQL Server** must be installed. [Download ODBC Driver](https://go.microsoft.com/fwlink/?linkid=2155042)
- Have your SQL Server address, a username and password with rights to list databases and create tables, and permission to insert data.
- You should have exported your Kobo survey form as JSON, and your Kobo survey data as CSV, XLSX, or JSON.

---

## Step-by-Step Usage Instructions

### 1. **Launch the Application**
   - Double-click `kobo_sql_mapper.exe` in this folder.

### 2. **Connect to SQL Server**
   - Enter your SQL Server **address**, **username**, and **password**.
   - Click ‘Connect to SQL Server’.
   - If successful, a dropdown with all user databases will appear.

### 3. **Select the Target Database**
   - Choose the database you want to use (your destination) from the dropdown.

### 4. **Upload Kobo Survey Form JSON**
   - Click **‘Upload Form JSON’** and select your Kobo XLSForm export (.json).
   - The app will read the question/column names for the future table.

### 5. **Specify Table Name**
   - Enter your desired table name, or accept the default (`kobo_survey`).

### 6. **Generate and Review CREATE TABLE SQL**
   - Click **‘Generate CREATE TABLE SQL’**.
   - The SQL will appear in the text area.
   - **Copy this SQL, and run it in SQL Server Management Studio (SSMS) or on Azure** to create the table.
   - The SQL statement will also be saved as `create_table.sql` next to this README for your reference.

### 7. **Confirm Table Exists**
   - Once you have created the table, click **‘Check Table Exists’**.
   - The application will verify that the table is present in your selected database.
   - When confirmed, the options to upload and map data will become active.

### 8. **Upload Survey Data**
   - Click **‘Upload Data File’** and choose your Kobo submissions or exports (formats: `.csv`, `.xlsx`, `.json` with column names matching your survey questions).

### 9. **Map Kobo Data to SQL**
   - Click **‘Map Kobo Data to SQL’**.
   - All matched columns will be inserted into your selected table.
   - A success message will show how many records were uploaded.

---

## Troubleshooting

- **Cannot Connect to SQL Server:** Check the address, credentials, server reachability, and that you have ODBC Driver 17 installed.
- **No Databases Listed:** Ensure your user account has permission to list/access databases on the server.
- **Table Does Not Appear:** Confirm you copied and ran the SQL exactly as generated.
- **Column Mismatches:** The tool only uploads columns that match both KoBo data and your table. Unmatched columns are ignored.

## FAQ

**Q: Can I use with Azure SQL?**  
A: Yes! Enter the Azure SQL server address and credentials.

**Q: Can I use my preferred table name?**  
A: Yes, enter any valid SQL table name. You’re no longer required to use a fixed name.

**Q: Do I have to run the SQL in SSMS/Azure first?**  
A: Yes, this step ensures full control. You can review and edit the CREATE TABLE SQL before proceeding.

**Q: Can I run the mapping/upload multiple times?**  
A: Yes, but the app will not overwrite existing records. Uploads are appended.
